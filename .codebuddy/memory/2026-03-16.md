## Bug 修复：我的课程页面显示不稳定

### 问题
用户反馈:"感觉有BUG。有时候不能显示 有时候可以显示"

1. **缺少 loading 状态反馈** - 用户不知道数据是否在加载
2. **云开发查询方式错误** - 使用 `where` 查询可能因权限或索引问题失败
3. **缺少订单更新事件** - 购买后没有自动刷新"我的课程"页面
4. **异步加载时序问题** - 页面渲染时数据可能还未加载完成

- ✅ 添加用户登录状态检查
- ✅ 添加详细的调试日志

**文件: `src/services/CloudOrderService.ts`**
- ✅ **关键修复**: 改用 `getAll('orders')` 替代 `where` 查询
- ✅ 云开发数据库会自动根据 `_openid` 过滤用户数据
- ✅ 添加详细的控制台日志

**文件: `src/pages/Checkout.tsx`**
- ✅ 在支付成功后触发 `ordersUpdated` 事件

### 修复结果

- 加载失败时会显示错误提示和重试按钮
- 云开发查询更稳定,利用自动过滤机制

### 功能特性

**学习进度服务 (`CloudProgressService`)**:
1. **获取课程进度** - `getCourseProgress(courseId)`
2. **获取所有进度** - `getAllProgress()`
3. **保存/更新进度** - `saveProgress(data)`
4. **标记课时完成** - `markLessonComplete(courseId, lessonId, totalLessons)`
5. **更新当前课时** - `updateCurrentLesson(courseId, lessonId)`
6. **删除进度** - `deleteProgress(courseId)`

**学习页面 (`LessonView`)**:
- 自动保存当前课时位置
- 完成课时自动计算进度
- 手动保存按钮支持
- 保存成功/失败提示
- 课时完成状态可视化
- 进度百分比实时显示

**我的课程页面 (`MyCourses`)**:
- 显示每门课程的学习进度条

- 已完成课程显示"已完成"标签

### 数据库设计

**user_progress 集合**:
```json
{
  "_id": "文档ID",
  "_openid": "用户OpenID(自动)",
  "userId": "用户ID",
  "courseId": "课程ID",
  "currentLessonId": "当前课时ID",
  "completedLessons": ["已完成的课时ID"],
  "progress": 85,  // 0-100
  "isCompleted": false,
  "lastStudyTime": "最后学习时间",
  "createdAt": "创建时间",
  "updatedAt": "更新时间"
}
```

**安全规则**:
```json
{
  "read": "auth.openid == doc._openid",
  "write": "auth.openid == doc._openid"
}
```
