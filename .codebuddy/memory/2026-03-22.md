```typescript
// src/utils/cloudbase.ts
export const ensureAuthenticated = async () => {
  const auth = app.auth();
  try {
    const { data } = await auth.getSession();
    if (data.session) return data.session;
  } catch (e) {}
  await auth.signInAnonymously();
  return data.session;
};
```

#### 修复 3：在数据库操作前调用自动登录
```typescript
// src/services/teacherServiceDirect.ts
async create(data: any) {

  const result = await db.collection('teacher_profiles').add(data);
  // ...
}

async update(teacherId: string, data: any) {
  await ensureAuthenticated(); // 确保已登录
```

## ✅ 修复数据库 create 函数返回值处理

**时间**: 2026-03-22 13:15

### 问题分析

**错误信息**: "Cannot read properties of undefined (reading 'id')"

**根本原因**: 
- CloudBase SDK 的 `add()` 方法返回格式不确定
- 有时返回 `{ id: string }`
- 有时返回 `{ _id: string }`
- 代码只处理了 `result.id` 的情况

**修复方案**:
1. 在 create 函数中同时检查 `result.id` 和 `result._id`
2. 使用 `result.id || result._id` 确保获取到 ID
3. 在调用 create 后验证返回值

**修改文件**:
- `src/services/database.ts`
  - create 函数: 添加兼容性处理 `result.id || result._id`

**代码修复**:
```typescript
// ❌ 修复前
const { data: result } = await db.collection(collectionName).add(doc);
return { _id: result.id, ...doc } as T;

// ✅ 修复后
const result = await db.collection(collectionName).add(doc);
return { _id: result.id || result._id, ...doc } as T;
```
