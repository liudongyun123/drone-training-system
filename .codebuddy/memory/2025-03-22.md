# 2025-03-22 工作日志

## 管理后台权限错误修复 - 完整解决

### 问题
管理后台所有模块功能无法使用,显示 `PERMISSION_DENIED` 错误。

### 根本原因
系统混合使用了两种数据访问方式:
- 直接 CloudBase SDK 调用 (正常工作)
- 云函数调用 (权限问题导致错误)

### 解决方案
将所有服务统一改为直接使用 CloudBase SDK 调用数据库。

### 完成的工作

#### 1. 创建新的直接 SDK 服务
- `scheduleServiceDirect.ts` - 排课管理服务
- `financeServiceDirect.ts` - 财务统计服务  
- `enrollmentServiceDirect.ts` - 报名和出勤管理服务

#### 2. 更新管理后台页面
- `AdminSchedules.tsx` - 使用 scheduleServiceDirect 和 attendanceServiceDirect
- `AdminFinance.tsx` - 使用 financeServiceDirect
- `AdminStudents.tsx` - 使用 enrollmentServiceDirect 和 attendanceServiceDirect

#### 3. 数据库权限
- 已确认所有相关集合使用 CUSTOM 权限: `{"read": true, "write": true}`
- 涵盖 collections: orders, enrollments, schedules, attendance_records, teachers 等

#### 4. 构建和部署
- 构建命令: `npm run build` ✅
- 部署命令: `npx tcb hosting deploy dist` ✅
- 部署地址: https://rcwljy-5ghmq2ex26764978-1318564729.tcloudbaseapp.com

### 架构改进
```
之前: 前端页面 → 云函数 → 数据库 (PERMISSION_DENIED)
现在: 前端页面 → CloudBase SDK → 数据库 (✓ 直接访问)
```

### 优势
1. 减少依赖,降低故障点
2. 提高性能,减少网络跳转
3. 简化调试,错误信息更清晰
4. 降低成本,减少云函数调用次数
5. 更好的类型安全
