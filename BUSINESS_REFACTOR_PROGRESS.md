## 核心变更

### 1. 数据模型关系变更

| 原逻辑 | 新逻辑 |
|--------|--------|
| 课程(Course)直接关联排课 | 课程(Course)只包含视频内容 |
| 报名(Enrollment)关联课程 | 报名(Registration)关联班级(Class) |
| 排课独立存在 | 排课从属于班级 |

### 2. 新的数据流向

```
线下报名流程:
学员浏览课程 → 选择班级 → 提交报名 → 审核通过 → 分配班级 → 查看课表
                                                          ↓
                                               视频权限(可选开通)

线上购买流程:
学员浏览课程 → 在线支付 → 立即开通视频权限 → 视频学习 → 在线考试
```

## 业务逻辑总结

### 新的数据流：

```
线下培训流程：
课程详情页 → 选择班级 → 提交报名 → 等待审核 → 分配班级 → 开通视频权限

线上课程流程：
课程详情页 → 立即购买/加入购物车 → 支付 → 直接开通视频权限
```

### 权限控制：
- 视频观看权限通过 `registration.videoAccess.hasAccess` 控制
- 班级课表通过 `class_schedules` 集合管理
- 调课通过 `transfer_requests` 集合管理

## 关键业务规则

### 1. 班级容量管理
- 班级有最大容量限制
- 报名审核时检查容量
- 满员后自动停止报名

### 2. 视频权限控制
- 班级可配置是否开通视频权限
- 视频有效期可独立设置
- 支持按报名记录单独控制

### 3. 调课流程
- 管理员可在班级内调整排课
- 调课自动通知学员
- 保留调课历史记录
