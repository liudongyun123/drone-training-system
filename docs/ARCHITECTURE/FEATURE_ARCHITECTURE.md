## 📁 目录结构

```
src/
├── features/                     # 🌟 Feature 模块
│   ├── course/                   # 课程功能模块
│   │   ├── types/               # 类型定义
│   │   ├── api/                 # API 接口
│   │   ├── hooks/               # 业务 Hooks
│   │   ├── components/           # 业务组件
│   │   └── pages/               # 页面组件
│   ├── class/                   # 班级模块
│   ├── order/                   # 订单模块
│   └── user/                    # 用户模块
├── platform/                     # 🌟 Platform Adapter
│   ├── adapters/                # 平台适配器
│   └── hooks/                   # 平台 Hooks
├── infrastructure/               # 🌟 基础设施层
│   ├── cache/                   # 缓存系统

│   ├── monitor/                 # 监控系统
│   └── errors/                  # 错误处理

```

## 🔧 Feature 模块标准结构

每个 Feature 必须包含 5 个部分：

```typescript
// 1. types/ - 类型定义
export interface Course {
  id: string;
  title: string;
}

// 2. api/ - API 接口
export const courseApi = {
  getList: (params) => { /* ... */ },
};

// 3. hooks/ - 业务逻辑
export const useCourseList = (params) => {
  // 组合 API + 缓存 + 错误处理
};

// 4. components/ - 展示组件
export const CourseCard = ({ course }) => {
  // 纯展示
};

// 5. pages/ - 页面组件
export const CourseListPage = () => {
  const { courses } = useCourseList();
  return <CourseList courses={courses} />;
};
```
