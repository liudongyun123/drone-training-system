# 数据库集合创建操作步骤

## 📋 需要创建的 9 个集合

1. user_profiles（学员档案）
2. teacher_profiles（教师档案）
3. course_schedules（课程排课）
4. enrollments（报名记录）
5. schedule_changes（调课申请）
6. attendance_records（出勤记录）
7. promotions（宣传素材）
8. statistics_daily（每日统计）
9. statistics_teacher（教师统计）

## ⚠️ 重要提示

**CloudBase 不允许通过代码自动创建集合，必须手动在控制台创建！**

控制台页面已经打开，请按照以下步骤操作。

---

## 🎯 操作步骤

### 步骤一：创建集合

1. 在已打开的控制台页面，点击右上角的 **"新建集合"** 按钮

2. 输入集合名称，例如：`user_profiles`

3. 权限选择：**"仅创建者和管理员可读写"（PRIVATE）**

4. 点击 **"确定"** 按钮

5. 等待集合创建成功

6. 重复以上步骤，为其余 8 个集合创建

---

### 步骤二：配置安全规则

集合创建后，需要配置安全规则。根据不同的数据类型，选择不同的权限类别。

#### 权限类别说明

| 权限类别 | 说明 |
|---------|------|
| 所有用户可读，仅创建者和管理员可写 | 公开数据（课程、轮播图、公告） |
| 仅创建者和管理员可读写 | 私有数据（购物车、用户设置） |
| 所有人可读，仅管理员可写 | 管理员管理的数据（教师档案、排课） |
| 仅管理员可读写 | 敏感数据（统计报表） |

#### 各集合推荐权限

| 集合名称 | 推荐权限 | 说明 |
|-----------|---------|------|
| user_profiles | 仅创建者和管理员可读写 | 学员个人隐私数据 |
| teacher_profiles | 所有用户可读，仅创建者和管理员可写 | 教师信息公开查看 |
| course_schedules | 所有人可读，仅管理员可写 | 所有人可看排课，仅管理员可改 |
| enrollments | 仅创建者和管理员可读写 | 报名记录个人隐私 |
| schedule_changes | 仅创建者和管理员可读写 | 调课申请个人隐私 |
| attendance_records | 所有人可读，仅管理员可写 | 教师可看出勤，仅管理员可改 |
| promotions | 所有人可读，仅管理员可写 | 仅管理员管理宣传素材 |
| statistics_daily | 仅管理员可读写 | 敏感统计数据 |
| statistics_teacher | 仅管理员可读写 | 敏感统计数据 |

#### 配置安全规则步骤

1. 点击集合名称进入集合详情页面

2. 点击 **"安全规则"** 标签

3. 点击 **"权限设置"** 或 **"编辑规则"** 按钮

4. 从下拉菜单选择推荐的权限类别

5. 点击 **"保存"** 按钮

## ❓ 常见问题

### Q: 为什么不能通过代码创建集合？
A: CloudBase 的安全策略要求所有集合必须在控制台手动创建，以确保安全性和权限控制。

### Q: 为什么要选择不同的权限类别？
A: 根据数据的敏感程度选择合适的权限：
- 个人隐私数据（学员档案、报名记录）→ PRIVATE
- 公开数据（教师信息、排课）→ READONLY 或 ADMINWRITE
- 管理数据（统计数据）→ ADMINONLY

### Q: 如果我选错了权限怎么办？
A: 可以在控制台重新编辑安全规则，选择不同的权限类别。

### Q: 规则生效后还是报错怎么办？
A:
1. 确认已等待 2-5 分钟
2. 检查用户是否已登录
3. 检查用户类型是否正确（管理员/学员/教师）
4. 查看浏览器控制台的错误信息

### Q: 如何测试权限是否生效？
A: 在浏览器控制台执行以下代码测试：
```javascript
import app from './config/tcb'
import { adminService } from './services/adminService'

// 测试读取集合
async function testRead() {
  try {
    const result = await adminService.list('user_profiles', {}, { limit: 1 })
    console.log('读取成功:', result)
  } catch (error) {
    console.error('读取失败:', error)
  }
}

testRead()
```
