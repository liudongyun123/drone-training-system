# 安全规则配置指南

## 重要说明

CloudBase 安全规则工具仅支持预设的权限类别，不支持完全自定义的 JSON 规则。

### 预设权限类别

| 权限类别 | 说明 | 适用场景 |
|---------|------|---------|
| `READONLY` | 所有人可读，仅创建者和管理员可写 | 公开数据（课程、轮播图、公告）|
| `PRIVATE` | 仅创建者和管理员可读写 | 私有数据（购物车、用户设置）|
| `ADMINWRITE` | 所有人可读，仅管理员可写 | 管理员管理的数据（教师档案、排课）|
| `ADMINONLY` | 仅管理员可读写 | 敏感数据（统计报表）|
| `CUSTOM` | 自定义规则 | 复杂权限控制（需在控制台配置）|

## 集合安全规则配置建议

### 1. user_profiles（学员档案表）
**推荐**：`PRIVATE`
- 原因：学员档案属于个人隐私数据
- 配置方式：在控制台设置"私有"
- 备选方案：控制台手动配置自定义规则

### 2. teacher_profiles（教师档案表）
**推荐**：`READONLY`
- 原因：教师信息需要所有人可查看
- 配置方式：设置"所有用户可读"

### 3. course_schedules（课程排课表）
**推荐**：`ADMINWRITE`
- 原因：所有人可查看排课，仅管理员可修改
- 配置方式：设置"仅管理员可写"

### 4. enrollments（报名记录表）
**推荐**：`PRIVATE`
- 原因：报名记录属于个人隐私数据
- 配置方式：设置"私有"
- 注意：实际业务中需要教师也能看到自己课程的报名记录

### 5. schedule_changes（调课申请表）
**推荐**：`PRIVATE`
- 原因：调课申请属于个人数据
- 配置方式：设置"私有"

### 6. attendance_records（出勤记录表）
**推荐**：`ADMINWRITE`
- 原因：出勤数据需要教师查看（统计），仅管理员可修改
- 配置方式：设置"仅管理员可写"

### 7. promotions（宣传素材表）
**推荐**：`ADMINWRITE` 或 `ADMINONLY`
- 原因：宣传素材由管理员管理
- 配置方式：设置"仅管理员可写"或"仅管理员可读写"

### 8. statistics_daily（每日统计表）
**推荐**：`ADMINONLY`
- 原因：统计数据仅管理员可访问
- 配置方式：设置"仅管理员可读写"

### 9. statistics_teacher（教师统计表）
**推荐**：`ADMINONLY`
- 原因：教师统计数据仅管理员可访问
- 配置方式：设置"仅管理员可读写"

## 控制台配置步骤

### 方法一：使用预设权限类别（推荐）

1. 打开数据库控制台
2. 点击集合名称进入集合详情
3. 点击"安全规则"标签
4. 点击"权限设置"或"编辑规则"
5. 从下拉菜单选择预设权限类别
6. 点击"保存"

### 方法二：使用自定义规则（精细控制）

1. 打开数据库控制台
2. 点击集合名称进入集合详情
3. 点击"安全规则"标签
4. 点击"编辑规则"
5. 粘贴自定义 JSON 规则
6. 点击"保存"

## 推荐的权限配置顺序

### 第一批：公开数据（所有人可读）
- teacher_profiles → `READONLY`
- course_schedules → `READONLY`
- attendance_records → `ADMINWRITE`

### 第二批：私有数据（仅创建者访问）
- user_profiles → `PRIVATE`
- enrollments → `PRIVATE`
- schedule_changes → `PRIVATE`

### 第三批：管理员数据（仅管理员访问）
- promotions → `ADMINONLY`
- statistics_daily → `ADMINONLY`
- statistics_teacher → `ADMINONLY`

## 注意事项

### 1. 安全规则生效时间
配置安全规则后，需要等待 **2-5 分钟**让规则生效。

### 2. 规则测试
配置完成后，建议在浏览器控制台测试数据访问：
```javascript
// 测试读取
import app from './config/tcb'
const db = app.database()

// 测试读取集合
const result = await db.collection('user_profiles').get()
console.log(result)
```

### 3. 规则冲突
如果使用了 `PRIVATE` 权限，管理员也无法直接访问该集合的数据。需要通过云函数 `admin` 来操作。

### 4. 云函数权限
云函数不受安全规则限制，可以读写所有集合的数据。这是为什么我们所有的写操作都通过云函数执行。

## 常见问题

### Q: 我可以使用你提供的 JSON 规则吗？
A: 可以，但需要在控制台手动粘贴到"自定义规则"中。MCP 工具只支持预设的权限类别。

### Q: 为什么推荐使用预设权限而不是自定义规则？
A: 预设权限更简单、更安全、更容易维护。自定义规则适合复杂的权限控制需求。

### Q: 如果我配置错了怎么办？
A: 可以在控制台重新编辑安全规则，或者选择不同的预设权限类别。

### Q: 管理员如何访问 PRIVATE 权限的集合？
A: 通过云函数 `admin` 访问。云函数有完整权限，不受安全规则限制。

### Q: 为什么 attendance_records 使用 ADMINWRITE 而不是 PRIVATE？
A: 因为教师需要查看自己课程的出勤记录，使用 ADMINWRITE 可以让所有登录用户读取，但只有管理员可以修改。
